# Deployment

Concord splits across two independently hosted pieces:

| Part | What it is | Suitable platforms |
|---|---|---|
| `concord-sync-server` | Node.js + Socket.io (WebSocket) | Render, Railway, Fly.io |
| `concord-text` | Static React + Vite build | Render, Vercel, Netlify, Cloudflare Pages |

> **Why split?** Vercel and Netlify serverless functions do not support long-lived WebSocket connections. The sync server must run as a persistent Node.js process.

---

## Prerequisites

- Node.js **22** (see `.node-version`)
- pnpm ≥ 9

---

## Option A — Render (recommended, one-command deploy)

Render can deploy both services and provision a PostgreSQL database from a single `render.yaml` in the repository root.

### Steps

**1. Connect your repository**

Go to [dashboard.render.com](https://dashboard.render.com) → **New** → **Blueprint** → select your `concord` repository. Render will detect `render.yaml` automatically.

**2. Update the placeholder URLs in `render.yaml`**

Before deploying, edit the two placeholder values:

```yaml
# In the sync-server service:
- key: CORS_ORIGIN
  value: https://concord-text.onrender.com   # ← set to your frontend URL

# In the text-editor service:
- key: VITE_API_URL
  value: https://concord-sync-server.onrender.com  # ← set to your backend URL
```

> Render assigns service URLs based on the service name. If you keep the default names (`concord-sync-server`, `concord-text`) the placeholder URLs above will be correct.

**3. Deploy**

Click **Apply**. Render will:
1. Provision a PostgreSQL 16 database (`concord-db`)
2. Build and deploy `concord-sync-server`
3. Build and deploy `concord-text`

`JWT_SECRET` is auto-generated by Render (`generateValue: true`).

**4. Initialize the database schema** *(PostgreSQL only)*

After the first deploy, open a **Shell** on the `concord-sync-server` service in Render and run:

```bash
psql $DATABASE_URL -f apps/sync-server/src/persistence/schema.sql
```

The SQLite fallback initializes itself automatically — no action needed.

## Option B — Railway (backend) + Vercel (frontend)

### Backend on Railway

1. Create a new Railway project and link your GitHub repository.
2. Add a **PostgreSQL** plugin — Railway injects `DATABASE_URL` automatically.
3. Set environment variables:

```
NODE_ENV=production
JWT_SECRET=<openssl rand -hex 32>
CORS_ORIGIN=https://your-frontend.vercel.app
PORT=3001
```

4. Set **Build command**:
```bash
pnpm install --frozen-lockfile && pnpm --filter concord-core run build && pnpm --filter concord-sync-server run build
```

5. Set **Start command**:
```bash
node --experimental-sqlite apps/sync-server/dist/index.js
```

6. Initialize the schema after first deploy:
```bash
railway run psql $DATABASE_URL -f apps/sync-server/src/persistence/schema.sql
```

### Frontend on Vercel

1. Import the repository into Vercel.
2. Set **Framework preset** to `Vite`.
3. Set **Root directory** to `apps/text-editor`.
4. Set environment variable:
```
VITE_API_URL=https://your-railway-backend.up.railway.app
```
5. Set **Build command**: `cd ../.. && pnpm --filter concord-core run build && pnpm --filter concord-text run build`
6. Set **Output directory**: `dist`

## Environment Variables Reference

### Sync server (production)

| Variable | Required | How to set |
|---|---|---|
| `NODE_ENV` | Yes | `production` |
| `DATABASE_URL` | Yes | Injected by DB plugin, or set manually |
| `JWT_SECRET` | **Yes** | `openssl rand -hex 32` |
| `CORS_ORIGIN` | Yes | Your frontend URL, no trailing slash |
| `PORT` | No | Platform-injected; defaults to `3001` |

See [`apps/sync-server/.env.production.example`](apps/sync-server/.env.production.example) for a ready-to-copy template.

### Frontend (build time)

| Variable | Required | Example |
|---|---|---|
| `VITE_API_URL` | Yes | `https://concord-sync-server.onrender.com` |

## Health Check

The sync server exposes `GET /health`:

```json
{ "status": "ok", "timestamp": "2024-07-18T10:30:00.000Z" }
```

Render uses this path automatically (configured in `render.yaml`). For other platforms, configure your health check to `GET /health`.

## Local Production Build Test

Verify the production build works before pushing:

```bash
# Full build (core → server → frontend)
pnpm build

# Test the server starts cleanly
node --experimental-sqlite apps/sync-server/dist/index.js

# In a separate terminal — preview the frontend build
pnpm --filter concord-text run preview
```

Expected output on server start:
```
{"level":30,"msg":"🚀 Concord sync server running on http://localhost:3001"}
{"level":30,"msg":"   WebSocket endpoint: ws://localhost:3001"}
```

If you see `ERR_UNKNOWN_BUILTIN_MODULE: node:sqlite`, your Node.js version is below 22. Check with `node --version`.
